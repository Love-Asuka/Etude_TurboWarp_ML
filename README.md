# Etude-TurboWarp-ML 使用文档

**版本**: 1.1.0  
**作者**: Asuka | Lin Xin  
**适用平台**: TurboWarp (Custom Extensions)

## 1. 简介
Etude-TurboWarp-ML 是一个为 TurboWarp 设计的轻量级机器学习扩展。它允许用户通过积木构建全连接神经网络（MLP），支持前向传播（推理）、反向传播（训练/自动微分）以及权重导出与加载。

**核心特性：**
*   支持多层线性层堆叠。
*   内置常用激活函数：ReLU, Tanh, Sigmoid, Softmax。
*   支持在线训练（SGD 优化器）。
*   标准化的 JSON 权重格式，方便模型保存与分享。

## 2. 积木详解

### 模型构建类

#### `构建: 添加线性层 [NAME] 输入 [IN] 输出 [OUT] 激活 [ACT]`
定义神经网络的一层。
*   **NAME**: 层的唯一名称（如 `fc1`）。
*   **IN**: 输入维度。
    *   *注意*: 只有第一层需要准确填写输入维度。后续层会自动忽略此参数，强制将输入维度对齐到上一层的输出维度。
*   **OUT**: 输出维度（神经元数量）。
*   **ACT**: 激活函数 (relu, tanh, softmax, sigmoid, none)。

#### `构建: 编译模型`
**必须执行**。在定义完所有层之后调用。此积木会生成计算图并初始化参数内存。在编译完成前无法进行传播。

#### `核心: 从JSON加载完整模型 [JSON]`
直接使用 JSON 字符串加载一个预定义好权重的模型。加载成功后无需再次编译，可以直接运行。
*   **JSON**: 符合 `turbowarp-nn-weights` 格式的字符串。

---

### 运行与训练类

#### `运行: 前向传播 输入 [INPUT]`
将数据输入模型进行预测。
*   **INPUT**: 输入数据的 JSON 字符串。
    *   格式示例: `"[1, 0, 0.5, -1]"` (向量) 或 `"[[1, 0, 0.5, -1]]"` (矩阵)。
*   **返回**: 输出结果的 JSON 字符串。

#### `训练: 反向传播 目标 [TARGET]`
计算误差并反向传播梯度（自动微分）。
*   **TARGET**: 期望的输出结果（标签）。
    *   格式示例: `"[0, 1]"`。
*   *机制*: 使用均方误差 (MSE) 计算损失，支持 Softmax 的正确梯度计算。

#### `训练: 应用梯度 学习率 [LR]`
根据反向传播计算出的梯度更新模型权重（随机梯度下降 SGD）。
*   **LR**: 学习率（Learning Rate），如 `0.01`。数值越小收敛越慢但越稳定。

---

### 数据与工具类

#### `数据: 获取当前输出`
获取最近一次前向传播的计算结果（JSON 格式）。

#### `数据: 导出模型JSON`
将当前模型的结构和所有权重导出为 JSON 字符串。你可以将此字符串保存到变量或文本文件中，以便下次通过 `加载权重` 积木恢复。

#### `工具: 矩阵乘法 [A] × [B]`
辅助积木，用于执行两个矩阵的乘法运算。

## 3. 快速上手示例

### 场景 A：从零开始训练一个简单的分类器
假设我们要训练一个网络，输入 `[1, 0]` 输出 `[1]`，输入 `[0, 1]` 输出 `[0]`。

1.  **初始化模型**
    *   `构建: 添加线性层 "L1" 输入 2 输出 4 激活 "relu"`
    *   `构建: 添加线性层 "L2" 输入 4 输出 1 激活 "sigmoid"`
    *   `构建: 编译模型`

2.  **训练循环 (重复执行 1000 次)**
    *   **样本 1**:
        *   `运行: 前向传播 输入 "[1, 0]"`
        *   `训练: 反向传播 目标 "[1]"`
        *   `训练: 应用梯度 学习率 0.1`
    *   **样本 2**:
        *   `运行: 前向传播 输入 "[0, 1]"`
        *   `训练: 反向传播 目标 "[0]"`
        *   `训练: 应用梯度 学习率 0.1`

3.  **测试**
    *   变量 `结果` = `运行: 前向传播 输入 "[1, 0]"`
    *   (此时 `结果` 应该非常接近 `[1]`)

---

### 场景 B：加载预训练权重
如果你已经有了导出的 JSON 权重数据。

1.  **加载**
    *   `核心: 从JSON加载完整模型 [变量: 权重JSON]`
2.  **推理**
    *   `说出 (运行: 前向传播 输入 "[0.5, -0.2, 1.0]")`

## 4. 数据格式说明

### 输入/输出格式
本扩展所有数据交互均基于 **JSON 字符串**。
*   **向量 (推荐)**: `"[0.1, 0.2, 0.3]"`
*   **单行矩阵**: `"[[0.1, 0.2, 0.3]]"`

### 权重 JSON 结构 (`turbowarp-nn-weights`)
导出的数据结构如下：
```json
{
  "format": "turbowarp-nn-weights",
  "version": "1.1",
  "model_meta": { ... },
  "layers": [
    {
      "type": "linear",
      "parameters": {
        "weight": { "shape": [8, 4], "data": [[...], [...]] }, // 权重矩阵
        "bias": { "shape": [8], "data": [...] }                 // 偏置向量
      },
      "activation": "relu"
      ...
    }
  ],
  "computation_graph": { ... }
}
```

## 5. 注意事项与最佳实践

1.  **Softmax 的使用**:
    *   Softmax 通常作为多分类任务的最后一层激活函数。
    *   如果你使用 Softmax，`输出维度` 应等于分类的类别数。
    *   反向传播时，目标 `TARGET` 应该是 One-hot 编码（例如 3 分类中的第 1 类应为 `[1, 0, 0]`）。

2.  **输入维度自动推断**:
    *   在堆叠层时，你只需要关心第一层的输入维度。
    *   例如：第一层输出 8，第二层的输入参数填多少都会被忽略，系统会自动设为 8。

3.  **性能提示**:
    *   由于是基于 JavaScript 在浏览器中运行，不建议构建过深（>10层）或过宽（>512神经元）的网络，否则可能会导致 Scratch 运行卡顿。
    *   前向传播是同步执行的。

4.  **数值稳定性**:
    *   扩展内部对 Softmax 做了防溢出处理。
    *   如果训练出现 `NaN` (Not a Number)，请尝试减小 `学习率` (例如从 0.1 改为 0.001)。
